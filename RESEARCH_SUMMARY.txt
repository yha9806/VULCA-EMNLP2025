性能指标（推荐方案：Sparticles.js）：
├─ 粒子数量：80-150个
├─ CPU占用：+3-5%（远低于可接受线）
├─ FPS保证：60fps桌面 / 30fps+移动
├─ 包体积：+10-15KB（仅为Chart.js的20%）
└─ 内存：+2-5MB（完全可接受）

【推荐库对比】━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

方案1：Sparticles.js ⭐⭐⭐⭐⭐（推荐）
  优点：性能最佳（9% CPU/1000粒子）| 极小包体（10KB） | 配置灵活 | CDN友好
  缺点：需要一些配置学习
  适用：当前平台的完美选择

方案2：CSS纯动画 ⭐⭐⭐⭐（保守选择）
  优点：零JS开销 | 极简 | 完全GPU加速
  缺点：限制<20粒子 | 不能交互
  适用：如果JS开销不可接受

方案3：PixiJS ⭐⭐⭐（未来装置）
  优点：100K+粒子 | GPU加速 | 高级功能
  缺点：250KB包体积 | 学习曲线陡
  适用：第3阶段（数据后现代装置）

【美学对齐分析】━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

当前设计"负形"（Negative Space）哲学：
  √ 极简 + 诗意 + 充足空白 + 有机动画

粒子如何增强而非破坏：

  传统极简主义：
    空白 = 虚空（纯无）

  +粒子后：
    空白 = 呼吸空间（为微妙动作）
    结果 = 虚空 + 存在的新张力 ✨

Sougwen Chung对齐：
  🔗 "人机协作" → 粒子代表AI的创意思维
  🔗 "拥抱错误" → 粒子的随机性 = 算法的优美不完美
  🔗 "过程可见" → 粒子 = 数字笔触，可见的思考过程

【三阶段路线图】━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🟢 第1阶段：英雄区域增强（立即）
   工作量：2-4小时
   范围：.hero 部分微妙粒子
   特点：80-150粒子 | 低密度 | 0.1-0.3透明度 | 缓慢漂浮
   风险：极低 ✓
   效果：增强标题的呼吸感，引入粒子美学

   配置示意：
   ┌─────────────────────────────────────────────────────┐
   │ new Sparticles(heroSection, {                         │
   │   count: 80,           // 低密度保留空白              │
   │   speed: 2,            // 缓慢（1-3范围）              │
   │   color: '#D4D2CE',    // 使用现有调色板              │
   │   minAlpha: 0.1,       // 幽灵般（0.1-0.3）            │
   │   maxAlpha: 0.3,                                      │
   │   drift: 2,            // 有机漂浮                     │
   │   twinkle: true,       // 诗意脉动                     │
   │   parallax: 0.5        // 轻微深度                     │
   │ })                                                    │
   └─────────────────────────────────────────────────────┘

🟡 第2阶段：RPAIT数据可视化（中优先级）
   工作量：1-2周
   范围：评论展示时的数据驱动粒子
   特点：粒子行为=RPAIT权重（表现力→数量 | 哲学→运动 等）
   风险：中等
   效果：RPAIT维度可感知，理解评论家的"声音"

   美学映射：
   表现力(R) ━→ 粒子数量    哲学家 = 多粒子、有机运动
   哲学(P)   ━→ 漂浮强度    技术者 = 少粒子、精确运动
   美学(A)   ━→ 颜色变化    审美家 = 色彩丰富
   解读(I)   ━→ 淡出模式    解读者 = 动态透明度
   技巧(T)   ━→ 运动速度

🔴 第3阶段：数据后现代装置（长期愿景）
   工作量：1-2个月
   范围：沉浸式PixiJS装置
   特点：100K+粒子 | 互动响应 | 艺术品数据可视化
   风险：高（但隔离新特性）
   效果：Refik Anadol风格体验，AI创意过程的可视化艺术

【风险与缓解】━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

风险                缓解方案
─────────────────────────────────────────────────────────
性能下降      →   监控CPU | 自适应粒子数 | 移动端禁用
美学冲突      →   第1阶段小范围测试 | 收集反馈
可访问性      →   尊重prefers-reduced-motion（内置）
CDN延迟       →   版本化URL（https://cdn.jsdelivr.net/...@1.6.0/...）
移动耗电      →   <768px自动禁用粒子
