# VULCA 前端重构完整方案
**项目名称**: 从网站到沉浸式艺术装置的升级
**设计哲学**: 精细打磨，视觉优先
**总周期**: 7周
**目标完成日期**: 2025年12月中旬

---

## 📋 方案概览

### 核心转变
```
当前状态 (网站)              目标状态 (艺术装置)
─────────────────────────────────────────────────
导航栏 / 菜单               → 全屏沉浸式Canvas
文本为主 / 线性流程          → 数据可视化 / 多维体验
静态内容 / 分页              → 实时动态粒子 / 同步展示
被动浏览 / 点击选择          → 主动探索 / 手势交互

结果：用户不是在"浏览网站"，而是在"体验艺术装置"
```

### 三个关键创新
1. **RPAIT驱动的粒子系统** (Phase 2)
   - 将评论权重转化为视觉语言
   - 24个粒子系统同时运行
   - 6位评论家的"声音"可视化

2. **沉浸式交互体验** (Phase 2 + Phase 3)
   - 悬停/点击/拖动改变粒子行为
   - 无导航菜单，纯视觉交互
   - 全屏Canvas + PixiJS高性能渲染

3. **数据后现代美学** (Phase 3)
   - Refik Anadol风格的数据绘画
   - 100K+粒子实时渲染
   - 用户交互 = 数据流可视化

## 🎨 关键设计决策

### 为什么是 2x2 网格?
- 平衡：足够显示4件完整作品
- 视觉焦点：每个作品独立呼吸空间
- 交互便利：4个区域易于精确悬停/点击

### 为什么是 PixiJS 而非 Three.js?
- 2D优化：不需要3D，2D更轻量
- 粒子性能：PixiJS ParticleContainer 专门优化
- 学习曲线：API 更直观，上手快

### 为什么粒子系统映射RPAIT?
- 数据 = 材料：粒子本身就是数据的视觉形式
- 6维度 = 6个视觉维度：完整映射，无信息丢失
- 实时响应：用户交互 → 权重变化 → 粒子变化（闭环）
